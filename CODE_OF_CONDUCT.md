# Code of Conduct

Lockroot is a security project. Discussion can be direct, technical, and critical, but it must stay respectful and useful.

## Expected Behavior

- Treat other contributors with respect.
- Critique code, design, and security assumptions without attacking people.
- Give clear reproduction steps when reporting bugs.
- Keep security discussions focused on responsible improvement.
- Assume good intent, but ask for clarification when something is unclear.

## Unacceptable Behavior

- Harassment, threats, abuse, or personal attacks.
- Discrimination or hateful language.
- Publicly posting private security reports before maintainers have had time to respond.
- Sharing real user secrets, vault files, passwords, or private keys.
- Spam, trolling, or unrelated promotion.
